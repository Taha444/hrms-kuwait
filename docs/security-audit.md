# تقرير المراجعة الأمنية والتقنية ومعالجة الثغرات

مراجعة شاملة للواجهة الخلفية والأمامية وقاعدة البيانات. كل بند مُعالَج ومُختبَر.

## الثغرات الأمنية المكتشفة والمُعالَجة

| # | الخطورة | الثغرة | الموقع | المعالجة |
|---|---|---|---|---|
| 1 | 🔴 حرجة | **كشف الملفات الحسّاسة (Public Uploads)**: مجلد `/uploads` كان يُقدَّم كملفات عامة بلا مصادقة — أي شخص يقرأ صور السيلفي والمستندات والشهادات | `main.py` | أُزيل التقديم العام؛ التنزيل الآن عبر نقاط موثّقة تتحقق من العزل والصلاحية |
| 2 | 🔴 حرجة | **Path Traversal / LFI**: مسار الـ SPA `/{full_path}` كان يقرأ أي ملف بالنظام عبر `../../` | `main.py` `serve_spa` | حلّ المسار والتحقق أنه داخل مجلد `dist` فقط؛ المسارات الخارجة تُعاد لـ index.html |
| 3 | 🟠 عالية | **Path Traversal في رفع الملفات**: استخدام `file.filename` مباشرة في المسار يسمح بالكتابة خارج مجلد الرفع (مستندات/طلبات/OCR) | `documents.py`، `requests.py` | `safe_files.unique_path` ينقّي الاسم ويولّد اسمًا فريدًا داخل المجلد فقط |
| 4 | 🟠 عالية | **تجاوز تغيير كلمة المرور**: `must_change_password` كان يُفرَض في الواجهة فقط — يمكن استخدام الـ API بكلمة المرور المؤقتة المشتركة | `deps.py` | فُرض على مستوى الخادم: يُرفض كل طلب (عدا /me و change-password) حتى التغيير |
| 5 | 🟡 متوسطة | **حقن HTML/XSS** في المستندات المولّدة (الصيغ وطلبات الإجازة/الراتب): بيانات الموظف والمدخلات تُدرَج بلا تهريب | `templates.py`، `workflow.py` | تهريب كل القيم المُدرجة عبر `html.escape` |
| 6 | 🟡 متوسطة | **استنزاف الذاكرة (DoS)**: لا حدّ لحجم الرفع | كل نقاط الرفع | حدّ أقصى 15MB عبر `read_limited` + امتدادات مسموحة فقط |
| 7 | 🟡 متوسطة | **حقن نوع المستند**: `kind` في رفع مستند الطلب كان حرًّا | `requests.py` | قائمة بيضاء للأنواع المسموحة |
| 8 | 🟢 منخفضة | **مقارنة غير ثابتة الزمن** لمفتاح شاشة الفرع (timing) | `kiosk.py` | `hmac.compare_digest` |
| 9 | 🟢 منخفضة | **مفتاح JWT افتراضي** قد يُستخدم في الإنتاج | `main.py` | تحذير عند الإقلاع إن كان المفتاح افتراضيًا |

## ضوابط أمنية مؤكَّدة سليمة (لا تحتاج تعديلًا)

- **حقن SQL**: كل الاستعلامات عبر SQLAlchemy ORM ببارامترات مربوطة — لا تكوين نصّي.
- **تجزئة كلمات المرور**: PBKDF2-SHA256 بـ240 ألف دورة + ملح عشوائي + مقارنة ثابتة الزمن.
- **JWT**: خوارزمية محدّدة (HS256) — لا قبول لـ `none` ولا خلط خوارزميات؛ تحقق من `type`.
- **العزل (Multi-Tenancy)**: مفروض على الخادم في كل نقطة (scope_company_id / assert_same_company) — مؤكَّد باختبارات.
- **القوة الغاشمة على الدخول**: قفل مؤقت بعد 5 محاولات فاشلة.
- **منع التعداد**: رسالة دخول موحّدة، ورسالة 403 موحّدة لمفتاح/فرع الشاشة.
- **CORS**: أصول محدّدة (لا `*`) في الإنتاج عبر `.env`.
- **رفع QR**: رموز موقّعة قصيرة + منع إعادة الاستخدام (anti-replay) عبر `jti`.
- **التسلسل الهرمي**: لا يدير مستخدم من هم في مستواه أو أعلى.

## مشاكل تقنية عُولِجت
- **تصادم أسماء** (`html` المتغيّر يحجب وحدة `html`) في توليد الصيغ — أُعيدت التسمية.
- **حدود الامتدادات** للملفات المرفوعة (قائمة بيضاء) لمنع رفع تنفيذيات.

## توصيات مستقبلية (ليست ثغرات حالية)
- تحديد معدّل عام للطلبات (rate-limiting) أمام الـ API (إضافة slowapi/بروكسي) — حاليًا القفل يكفي للدخول.
- إبطال/تدوير رموز التجديد (refresh token rotation) عند تسجيل الخروج.
- تحسين أداء بعض الاستعلامات (N+1) في مراجعة الحضور وتسلسل الطلبات عند التوسّع لآلاف الموظفين.
- تخزين الملفات في S3-compatible في الإنتاج بدل القرص المحلي.

## التحقق
- **36 اختبار pytest ناجح** — منها اختبارات أمنية: منع Path Traversal في الأسماء، فرض تغيير كلمة المرور على الخادم، وعدم كشف المرفوعات.
- تأكيد حيّ: محاولة `../../backend/.env` تُعاد لـ index.html (لا تسريب)، `/uploads` ترجع 404.
