# إعداد شاشة عرض QR للفروع (Kiosk) وتدفّق الحضور

## الفكرة باختصار
**الخادم هو مصدر الحقيقة الوحيد.** شاشة الفرع تجلب رمز QR موقّعًا من الخادم ويتجدّد كل
~90 ثانية. تطبيق الموظف يمسح الرمز ويُرسله للخادم للتحقق. الشاشة والموبايل لا يتكلّمان
مباشرة — كلاهما يتحقق عبر الخادم، فيبقيان متزامنين تلقائيًا.

```
شاشة الفرع ──GET /api/kiosk/{branch}/qr?key=──▶ الخادم ──يُصدر JWT(branch_id, jti, exp~90s)
الموظف يمسح الـ QR ──POST /api/attendance/validate-qr──▶ الخادم ──يتحقق ويُصدر checkin_ticket (~3 دقائق)
الموظف يلتقط السيلفي ──POST /api/attendance/check-in (ticket + selfie)──▶ الخادم ──يسجّل الحضور
```

## خطوات الإعداد (من لوحة الأدمن)
1. سجّل الدخول كمدير/مسؤول له صلاحية **إدارة الفروع** (`manage_branches`).
2. افتح صفحة **«الفروع وشاشات QR»**.
3. عند الفرع المطلوب اضغط **«توليد / تدوير مفتاح الشاشة»** → ينسخ لك رابط الشاشة:
   ```
   https://YOUR-DOMAIN/kiosk/qr/{branch_id}?key={kiosk_key}
   ```
4. افتح هذا الرابط على شاشة الفرع (تلفزيون ذكي/تابلت/ميني PC) في وضع **ملء الشاشة**.
   تبدأ بعرض الـ QR وتجدّده تلقائيًا للأبد — **بلا أي تحديث يدوي**.

> ⚠️ الكاميرا/GPS في المتصفح تتطلّب **HTTPS** (أو `localhost` للتطوير).

## التزامن والأمان
- **تزامن تلقائي:** الشاشة وأي موبايل يتحققان من نفس الخادم، فلا حاجة لمزامنة يدوية.
- **تدوير المفتاح:** لو تسرّب الرابط، اضغط «توليد / تدوير مفتاح الشاشة» — يبطل القديم فورًا.
- **منع إعادة الاستخدام (anti-replay):** كل رمز QR له `jti` يُستهلَك مرة واحدة، فلا يسجّل به
  موظفان بنفس اللقطة.
- **عدم فشل التسجيل عند دوران الرمز:** بعد التحقق يُصدَر `checkin_ticket` صالح ~3 دقائق، فلا
  يفشل التسجيل لو تغيّر الـ QR أثناء التقاط السيلفي.

## تدفّق الموظف (خطوتان)
1. **مسح الرمز:** يفتح الكاميرا الخلفية ويمسح QR الفرع (لا إدخال يدوي). يتحقق الخادم من
   التوقيع + الفرع + الأهلية + الـ Geofence.
2. **السيلفي:** تُفتح كاميرا الوجه **فقط بعد** نجاح المسح، ثم «تسجيل حضور/انصراف».
   لا يُقبل التسجيل بدون سيلفي.

## نقاط الـ API
| النقطة | الوصف |
|---|---|
| `GET /api/kiosk/{branch_id}/qr?key=...` | رمز الشاشة الحالي (عام، بالمفتاح فقط) |
| `POST /api/attendance/validate-qr` | `{qr_token, lat?, lng?}` → `{checkin_ticket}` |
| `POST /api/attendance/check-in` (multipart) | `{checkin_ticket, action: in\|out, selfie}` |
| `POST /api/branches/{id}/kiosk-key/rotate` | توليد/تدوير مفتاح الشاشة (أدمن) |
| `GET /api/branches/{id}/kiosk-url` | الرابط الحالي للشاشة (أدمن) |

## الحالات المرفوضة (تحقّق الخادم)
رمز منتهٍ/توقيع غير صالح · فرع غير مطابق · موظف غير مؤهَّل للفرع · خارج الـ Geofence ·
سيلفي مفقود · تذكرة مستخدَمة/منتهية · رمز QR مُعاد استخدامه.
