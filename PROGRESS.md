# سجل التقدّم (PROGRESS)

تنفيذ المشروع وفق المراحل المحددة في التكليف. كل مرحلة قابلة للتشغيل والاختبار.

| المرحلة | الوصف | الحالة |
|---|---|---|
| Phase 0 | الأساس: سكافولد monorepo، JWT + دخول بالرقم المدني + إجبار تغيير كلمة المرور + الأدوار السبعة + seed + i18n/RTL + Docker | ✅ مكتمل |
| Phase 1 | الشركات والموظفون: العزل، CRUD، تعدّد المديرين/HR/مسؤولي الفروع، نظام الصلاحيات الكامل، لوحات لكل دور | ✅ مكتمل |
| Phase 2 | خزنة المستندات بالنُسخ + «تنزيل الأحدث» + المسح اليومي المجدول + محرّك المهام متعدّد المستلِمين | ✅ مكتمل |
| Phase 3 | الحضور: الفروع + QR المتغيّر + السيلفي الإلزامي + GPS Geofence + الورديات + حساب التأخير/الإضافي + أنماط لكل موظف | ✅ مكتمل |
| Phase 3.1 (تحديث) | إعادة هيكلة تدفّق الحضور: **مسح QR ← ثم سيلفي** (wizard خطوتين، لا إدخال يدوي)، رموز QR موقّعة (JWT) قصيرة + تذكرة تسجيل + منع إعادة الاستخدام، و**صفحة عرض QR مستقلة للفروع (Kiosk)** بمفتاح قابل للتدوير | ✅ مكتمل |
| Phase 4 | محرّك الطلبات والموافقات: طلب الإجازة الكامل + شهادة الراتب + حق المدير في الإلغاء + صناديق «طلباتي/بانتظار موافقتي» + توليد/طباعة المستندات | ✅ مكتمل |
| Phase 5 | OCR: واجهة قابلة للتوصيل (جواز/بطاقة مدنية/عقود) + تأكيد المستخدم + مهمة «نقل المعلومات» التلقائية عند تجديد الجواز | ✅ مكتمل (مزوّد OCR فعلي يُوصَّل لاحقًا خلف الواجهة) |
| Phase 6 | الرواتب وEOS والتقارير: محرّك EOS مدموج ومُختبَر + خصومات/إجازات + لوحات | ✅ EOS مكتمل ومُختبَر · الرواتب أساسية |
| Phase 7 | التلميع والنشر: PWA (service worker + manifest + كاميرا/GPS) + docker-compose + تحصين أمني | ✅ مكتمل |

## معالجة النواقص (الدفعة الأخيرة)
| الناقص | الحالة |
|---|---|
| **مسيّر الرواتب** | ✅ محرّك كامل (أساسي + إضافي − غياب − خصومات) + حفظ المسيّرات + صفحة عرض |
| **التقارير والتصدير** | ✅ Excel + CSV (بدعم عربي/BOM) للموظفين والرواتب والحضور |
| **سجل التدقيق (UI)** | ✅ نقطة API + صفحة عرض مع فلترة |
| **إنهاء خدمة موظف** | ✅ يحسب EOS تلقائيًا ويؤرشف الحالة |
| **ترقيم الصفحات (Pagination)** | ✅ على قائمة الموظفين (limit/offset + X-Total-Count) |
| **التحقق من المدخلات** | ✅ الرقم المدني، القيم السالبة، نمط الحضور، منطق تواريخ الإجازة |
| **تحديد معدّل الدخول (Rate-limit)** | ✅ لكل IP (يُعطَّل في الاختبارات) |
| **OCR قارئ الجواز (MRZ)** | ✅ محلّل TD3 حقيقي مع التحقق من أرقام الضبط (ICAO 9303) |
| **قنوات واتساب/SMS** | ✅ واجهة قابلة للتوصيل + قناة Log فعّالة + منافذ Twilio جاهزة للضبط |

### تم توصيلها فعليًا (Gap Fix Plan V1.3.1)
- توليد PDF عربي أصيل من الخادم فعليًا (reportlab + خط Amiri + تشكيل/RTL) — لا يعتمد على طباعة المتصفح.
- محرّك OCR للصور (Tesseract عبر pytesseract) — يُكتشَف تلقائيًا إن توفّر الثنائي (مُثبَّت في Dockerfile)،
  ويتراجع بأمان لعدم التفعيل محليًا بدونه.
- إرسال واتساب/SMS فعلي عبر Twilio REST API (بلا مكتبة خارجية) — يُفعَّل بضبط `TWILIO_*` في `.env`.

### ما زال يحتاج إعدادًا خارجيًا
- بيانات اعتماد Twilio فعلية (Account SID/Auth Token/أرقام SMS و WhatsApp) لتفعيل الإرسال الحقيقي.
- اختبارات E2E للواجهة، CI/CD، وتخزين S3.

## ما تم اختباره
- `pytest` (14 اختبارًا): محرّك EOS بأمثلة القانون، العزل بين الشركات، فرض الصلاحيات على السيرفر، محرّك الطلبات end-to-end (إجازة كاملة + إلغاء المدير + شهادة راتب).
- بناء الواجهة الأمامية `npm run build` ينجح مع توليد service worker للـ PWA.
- تشغيل الخادم الحيّ: تسجيل دخول، لوحة تحكم، رمز QR الحيّ للفرع، المسح اليومي (توليد 7 مهام)، صندوق المهام.
- توليد هجرة Alembic الأولية والتحقق من `alembic upgrade head` على قاعدة جديدة.

## نقاط للتوسعة المستقبلية
- توصيل مزوّد OCR فعلي (MRZ/باركود) خلف `app/ocr.py`.
- توصيل قنوات واتساب/SMS خلف محرّك الإشعارات.
- توسيع مسيّر الرواتب وربط الخصومات/الإضافي تلقائيًا.
