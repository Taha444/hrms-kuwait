# نظام إدارة الموارد البشرية متعدد الشركات — دولة الكويت (HRMS / ERP)

نظام ويب متكامل (يُثبَّت كـ **PWA**) لإدارة الموارد البشرية لعدة شركات مستقلة على
**سيرفر مركزي واحد وقاعدة بيانات واحدة** مع **عزل تام بين الشركات (Multi-Tenancy)**.
مبني وفق قانون العمل الكويتي للقطاع الخاص رقم 6/2010.

> الواجهة الخلفية: **FastAPI + SQLAlchemy 2.0 + Alembic** · الواجهة الأمامية:
> **React + TypeScript + Vite (PWA, RTL, i18n)** · قاعدة البيانات: **PostgreSQL**
> (إنتاج) / **SQLite** (تطوير).

---

## ✨ أهم الميزات

- **مصادقة JWT** بالدخول عبر **الرقم المدني** + إجبار تغيير كلمة المرور لأول دخول + قفل بعد المحاولات الفاشلة + إعادة تعيين من الأدمن.
- **عزل تام بين الشركات** مفروض على مستوى السيرفر (لا يُعتمد على الواجهة).
- **نظام صلاحيات دقيق** (Permission-based) مع قوالب جاهزة، صلاحيات مؤقتة بتاريخ انتهاء، ونسخ صلاحيات.
- **سبعة أدوار**: إدارة عليا، صاحب شركة، مدير، مسؤول فرع، موارد بشرية، مندوب، وموظف (خدمة ذاتية).
- **خزنة مستندات بنُسخ (versioning)** + تنزيل الأحدث + واجهة OCR قابلة للتوصيل (جواز/بطاقة مدنية/عقود).
- **محرّك مهام وإشعارات** متعدّد المستلِمين + **مسح يومي مجدول** لانتهاء الإقامات/الجوازات/التراخيص + مهام متسلسلة (جواز جديد → نقل معلومات).
- **حضور وانصراف** بتدفّق آمن من خطوتين: **مسح رمز QR متغيّر** (الكاميرا الخلفية) ← تحقّق على الخادم ← **سيلفي إلزامي** (الكاميرا الأمامية تُفتح بعد التحقق فقط) ← تسجيل، مع تأكيد **GPS Geofence**، منع إعادة الاستخدام (anti-replay)، وحساب التأخير/الإضافي حسب الورديات. + **شاشة عرض QR مستقلة للفروع (Kiosk)** بلا تسجيل دخول — راجع [docs/kiosk-setup.md](docs/kiosk-setup.md).
- **محرّك طلبات وموافقات قابل للتهيئة**: طلب الإجازة الكامل (مسؤول الفرع → المدير → HR → طباعة → موعد توقيع → رفع الموقّع → إذن مغادرة المندوب → إشعار العامل) وطلب شهادة الراتب، مع حق المدير في الإلغاء بأي مرحلة.
- **حاسبة مكافأة نهاية الخدمة (EOS)** وفق قانون 6/2010 (مُختبَرة بالأمثلة).
- **لوحات تحكم لكل دور** + سجل تدقيق (Audit Log).

## 🚀 التشغيل السريع للتطوير (Windows / Linux / macOS)

### المتطلبات
- Python 3.11+
- Node.js 18+

### 1) الواجهة الخلفية
```bash
cd backend
python -m venv .venv
# تفعيل البيئة:  Windows: .venv\Scripts\activate  |  Linux/macOS: source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
python -m app.seed                 # تعبئة بيانات تجريبية
uvicorn app.main:app --reload --port 8001   # http://127.0.0.1:8001  (توثيق API: /docs)
# ملاحظة: المنفذ 8000 محجوز على بعض أجهزة ويندوز (Hyper-V/WSL)، لذا نستخدم 8001.
```

### 2) الواجهة الأمامية
```bash
cd frontend
npm install
npm run dev                        # http://localhost:5173  (بروكسي إلى 8000)
```

افتح <http://localhost:5173> وسجّل الدخول بأحد الحسابات أدناه.

## 🔐 حسابات تجريبية (الرقم المدني / كلمة المرور)

| الدور | الرقم المدني | كلمة المرور |
|---|---|---|
| إدارة عليا (super_admin) | `000000000000` | `admin123` |
| صاحب الشركات (company_owner) | `111111111111` | `owner123` |
| مدير شركة 1 (company_manager) | `100000000001` | `manager123` |
| شؤون الموظفين (hr) | `100000000002` | `hr12345` |
| مسؤول فرع حولي (branch_supervisor) | `100000000003` | `sup12345` |
| المندوب (delegate) | `100000000004` | `deleg123` |
| مدير شركة 2 | `200000000001` | `manager123` |
| عامل (خدمة ذاتية) | `289010112345` | `emp12345` |

> الحسابات التجريبية لا تُجبر على تغيير كلمة المرور لتسهيل التجربة؛ المستخدمون الجدد يُجبرون.

تغطي: محرّك EOS (أمثلة القانون)، العزل بين الشركات، فرض الصلاحيات، ومحرّك الطلبات end-to-end.

## 🐳 النشر بـ Docker (أمر واحد)

```bash
cp .env.example .env      # عدّل الأسرار
docker compose up --build
# الواجهة على http://localhost  — الـ API خلف /api
```
الخدمات: `db` (PostgreSQL) + `api` (FastAPI، يطبّق الهجرات تلقائيًا) + `web` (nginx يقدّم الواجهة ويمرّر /api).
للحصول على HTTPS راجع [docs/deployment.md](docs/deployment.md).

## 📌 ملاحظات
