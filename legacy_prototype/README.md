# 🏢 نظام إدارة الموارد البشرية متعدد الشركات — نسخة دولة الكويت

نظام ويب متكامل لإدارة عدة شركات مستقلة داخل منصة واحدة، مع **عزل كامل للبيانات** بين الشركات، ونظام **صلاحيات دقيق** (Permission-based)، ومحرك **مكافأة نهاية الخدمة** مُهيّأ وفق **قانون العمل الكويتي رقم 6 لسنة 2010** للقطاع الخاص.

---

## ⚙️ المتطلبات

- **Python 3.10** أو أحدث ([python.org](https://www.python.org/downloads/)).
  - تأكّد أثناء التثبيت على ويندوز من تفعيل خيار **Add Python to PATH**.
- لا حاجة لأي قاعدة بيانات خارجية — النظام يستخدم **SQLite** المدمج.
- لا حاجة لاتصال إنترنت بعد تثبيت Flask.

---

## 🚀 طريقة التشغيل على جهازك

### الطريقة الأسهل (سكربت جاهز)

**على ويندوز:** انقر نقرًا مزدوجًا على ملف `run.bat` (أو شغّله من موجّه الأوامر).

**على لينكس / ماك:** من الطرفية داخل مجلد المشروع:

```bash
chmod +x run.sh
./run.sh
```

السكربت يقوم تلقائيًا بـ: إنشاء بيئة افتراضية، تثبيت Flask، تهيئة قاعدة البيانات بالبيانات التجريبية، ثم تشغيل النظام.

ثم افتح المتصفح على العنوان: **http://127.0.0.1:5000**

---

### الطريقة اليدوية (خطوة بخطوة)

افتح الطرفية (Terminal / CMD) **داخل مجلد المشروع** ونفّذ:

```bash
# 1) إنشاء بيئة افتراضية
python -m venv venv

# 2) تفعيل البيئة الافتراضية
#    على ويندوز:
venv\Scripts\activate
#    على لينكس / ماك:
source venv/bin/activate

# 3) تثبيت المتطلبات
pip install -r requirements.txt

# 4) تهيئة قاعدة البيانات + بيانات تجريبية (مرة واحدة فقط)
python seed.py

# 5) تشغيل النظام
python app.py
```

بعد ظهور رسالة التشغيل، افتح المتصفح على: **http://127.0.0.1:5000**

> لإيقاف الخادم اضغط `Ctrl + C` في الطرفية.

## 🧩 المزايا المنفّذة

- **إدارة الشركات:** إضافة/تعديل/تفعيل/تعطيل/أرشفة، ولوحة تحكم مستقلة لكل شركة، ونقل موظف بين شركتين مع سجل تاريخي.
- **العزل التام (Multi-Tenancy):** كل كيان مرتبط بشركة، ولا يرى أي مستخدم بيانات شركة أخرى إلا الإدارة العليا.
- **نظام صلاحيات Action-based:** ١٦ صلاحية مستقلة، قوالب جاهزة، نسخ صلاحيات من مستخدم لآخر، وصلاحيات مؤقتة بتاريخ انتهاء.
- **الموظفون:** بيانات كاملة، الإقامات وأذونات العمل، الخصومات، الإجازات (طلب/موافقة/رفض)، وإنهاء الخدمة.
- **التراخيص:** CRUD كامل، ربط الموظفين، ومقارنة العمالة الفعلية بالحد المسموح مع تنبيه التجاوز.
- **مكافأة نهاية الخدمة:** محرك حساب دقيق وفق القانون الكويتي (تفاصيل أدناه)، مع حاسبة تفاعلية.
- **التنبيهات:** قرب/تجاوز انتهاء الإقامات والتراخيص والمستندات، وتجاوز سعة العمالة.
- **التقارير:** ملخص رقمي + تصدير CSV (بترميز يدعم العربية في Excel).
- **سجل التدقيق (Audit Trail):** تسجيل كل العمليات الحساسة.
- **الأمان:** كلمات مرور مُجزّأة (hashed)، جلسات موقّعة، حظر مؤقت بعد محاولات فاشلة، **حماية CSRF**، **رؤوس HTTP أمنية (CSP/X-Frame-Options)**، **حدّ معدّل الطلبات**، **خمول الجلسة**، التحقق من نوع الملفات المرفوعة، و**مصادقة ثنائية (2FA / TOTP)** اختيارية.

### مزايا أُضيفت في الإصدار 2.0

- **الأقسام والهيكل التنظيمي**، و**التاريخ الوظيفي** (ترقيات/تغيّر راتب) تلقائيًا.
- **الحضور والانصراف** + الساعات الإضافية.
- **مسيّر الرواتب (Payroll):** راتب أساسي + بدلات + إضافي − خصومات − تأمينات، مع قسائم رواتب.
- **رصيد الإجازة السنوية** يُحتسب تلقائيًا (افتراضي 30 يومًا) ويُخصم عند الاعتماد.
- **حساب وتخزين مكافأة نهاية الخدمة تلقائيًا** عند إنهاء الخدمة، و**تقرير إجمالي التزامات نهاية الخدمة**.
- **الجزاءات التأديبية، العُهد والأصول، تقييم الأداء**.
- **مركز إشعارات داخل التطبيق** + بريد إلكتروني/SMS اختياري + **فحص يومي مجدول** للتنبيهات.
- **بوابة الخدمة الذاتية** للموظف (عرض الملف/القسائم وطلب إجازة).
- **تصدير Excel و PDF** (شهادة راتب، مخالصة نهاية الخدمة) إضافةً إلى CSV.
- CRUD كامل (حذف/تعديل) للإقامات والخصومات والمستندات والموظفين.

## ⚖️ ملاحظات قانونية حول مكافأة نهاية الخدمة

المحرك مبني وفق **قانون العمل الكويتي رقم 6/2010 (القطاع الخاص) — المواد 51 إلى 53**:

- **أجر اليوم** = الراتب الشهري ÷ **26** (المعتمد في أغلب الحسابات). المقسوم قابل للتغيير لكل شركة (26 أو 30) حسب سياستها.
- **أول 5 سنوات:** 15 يومًا عن كل سنة. **ما بعد 5 سنوات:** 30 يومًا (شهر) عن كل سنة. والكسور تُحسب بالتناسب.
- **الحد الأقصى للمكافأة:** راتب 18 شهرًا.
- **نسبة الاستحقاق (عقد غير محدد المدة):**
  - فصل (غير تأديبي) / انتهاء العقد / وفاة / عجز / استقالة المرأة للزواج خلال سنة → **استحقاق كامل**.
  - استقالة: أقل من 3 سنوات → **0%**، من 3 لأقل من 5 → **50%**، من 5 لأقل من 10 → **66.67%**، 10 فأكثر → **100%**.
  - فصل تأديبي (المادة 41) → **0%**.
- يُضاف **بدل رصيد الإجازات** غير المستخدمة = أجر اليوم × عدد الأيام.

> ⚠️ **تنويه:** النتيجة تقديرية لأغراض النظام وليست استشارة قانونية. تُحسب على الراتب الأساسي ما لم ينص العقد على غير ذلك. للحالات الخاصة يُرجى مراجعة **الهيئة العامة للقوى العاملة (PAM)**.

## ⚙️ الإعدادات (متغيّرات البيئة)

تُقرأ الإعدادات من ملف `.env` (انسخ `.env.example` إلى `.env`). أهمها:

| المتغيّر | الوصف | الافتراضي |
|---------|------|----------|
| `HRMS_DEBUG` | وضع التطوير | `false` |
| `HRMS_SECRET_KEY` | المفتاح السري (عيّنه في الإنتاج) | يُولّد تلقائيًا |
| `HRMS_COOKIE_SECURE` | كوكي عبر HTTPS فقط | `false` |
| `HRMS_CSRF` | تفعيل حماية CSRF | `true` |
| `HRMS_PASSWORD_MIN_LEN` | أدنى طول لكلمة المرور | `8` |
| `HRMS_MAIL_ENABLED` / `HRMS_MAIL_*` | إشعارات البريد | معطّل |
| `HRMS_SMS_ENABLED` / `HRMS_SMS_*` | إشعارات SMS | معطّل |
| `HRMS_SCHEDULER_ENABLED` | الفحص اليومي المجدول | `false` |

```bash

```

## 🛡️ قبل النشر في بيئة إنتاج

1. غيّر **كل كلمات المرور** الافتراضية (تُجبَر على التغيير عند أول دخول للمستخدمين الجدد).
2. عيّن `HRMS_SECRET_KEY` ثابتًا وقويًا في `.env`.
3. اضبط `HRMS_DEBUG=false` و`HRMS_COOKIE_SECURE=true` (خلف HTTPS).
4. شغّل خلف `waitress` (عبر `wsgi.py`) أو Docker.
5. فعّل `HRMS_SCHEDULER_ENABLED=true` للفحص اليومي، وجدوِل `backup.py`.
